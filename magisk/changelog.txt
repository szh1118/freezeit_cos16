### v3.3.4SelfUse

- 修复 Binder 调试节点不可读时 `binder_queue unknown` 被误判为活动事务，所有候选应用永久停在等待冻结状态的问题；只有明确的 active/busy Binder 证据才延迟冻结。
- 修复 Binder freezer 降级后的 SIGSTOP 回退操作被错误记录为 `cgroup.freeze`；日志现在报告实际执行的 `signal.stop` 后端。
- 强化控制循环的冻结/解冻状态回收、重启恢复、前台保护、共享 UID 与系统 UID 防护，避免应用状态长期失真或错误控制。
- 加固 manager socket、Xposed bridge 与配置同步的长度限制、时限、会话一致性和失败降级路径。
- 修复 hook 侧前台、待冻结、WakeLock、Alarm、ANR、广播与 Athena 处理中的状态边界问题。
- Android 16 上未实现的 `BroadcastQueueModernImpl` 广播抑制按可选能力降级，不再关闭核心冻结/解冻控制；该广播拦截优化暂不可用。
- 修复管理器配置、缓存、网络、内容 URI 和页面并发更新中的资源泄漏、错误响应与 UI 回归。
- 发布链增加 APK 证书、构建会话、来源证明与 ZIP 校验覆盖；正式包只包含受控构建的 ARM64 Rust daemon 与管理器 APK。

### v3.3.3SelfUse

- 修复 manager socket 校验长度前按声明值分配内存且无读取超时；畸形长度不再触发 4GB 分配或阻塞控制循环。
- daemon 接受循环增加 panic 隔离，单个请求崩溃不再拖垮整个守护进程。
- 守护进程重启时补发 SIGCONT 恢复上一轮用信号冻结的进程，避免重启后应用永久卡死。
- 修复仅有 cgroup 无 binder freezer 时误走 Freeze 路径回滚、失去精确冻结的问题。
- 把 hook 健康检查网络往返移出 state 锁，慢 hook 桥接不再阻塞冻结/解冻。
- 修复 Xposed 桥接未捕获 IOException 导致重试耗尽、整周期桥接死亡的问题。
- 修复 Java 客户端 payloadLen 无上限校验导致 OOM 崩溃。
- 修复 Config 页下拉刷新后台线程 requireContext 崩溃。
- 修复 ClearLog 无法清屏、GetProcState 受日志级别过滤无响应。
- 设置写入改原子写，避免断电损坏；JSON 转义控制字符；安装器 APK 权限收紧为 0600。
