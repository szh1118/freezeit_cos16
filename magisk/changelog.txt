### v3.3.3SelfUse

- 修复 manager socket 校验长度前按声明值分配内存且无读取超时；畸形长度不再触发 4GB 分配或阻塞控制循环。
- daemon 接受循环增加 panic 隔离，单个请求崩溃不再拖垮整个守护进程。
- 守护进程重启时补发 SIGCONT 恢复上一轮用信号冻结的进程，避免重启后应用永久卡死。
- 修复仅有 cgroup 无 binder freezer 时误走 Freeze 路径回滚、失去精确冻结的问题。
- 把 hook 健康检查网络往返移出 state 锁，慢 hook 桥接不再阻塞冻结/解冻。
- 修复 Xposed 桥接未捕获 IOException 导致重试耗尽、整周期桥接死亡的问题。
- 修复 Java 客户端 payloadLen 无上限校验导致 OOM 崩溃。
- 修复 Config 页下拉刷新后台线程 requireContext 崩溃。
- 修复 ClearLog 无法清屏、GetProcState 受日志级别过滤无响应。
- 设置写入改原子写，避免断电损坏；JSON 转义控制字符；安装器 APK 权限收紧为 0600。

### v3.3.2SelfUse

- 修复 Manager 无操作保存时写入展示默认项的问题。
- 保留 `appcfg.txt` 原有包名、顺序、注释、空行与未修改内容。
- 修复冻结状态页因 daemon 缺少命令 11 而始终空白的问题。
- 相同共享 UID 进程集合的安全拒绝只记录一次，避免每秒刷屏。
- 下载延迟持续续期时只记录首次事件，避免重复刷屏。
- 将通用控制失败文案从误导性的 signal 失败改为控制应用失败。
- 清空工作日志前增加确认框，避免误触丢失日志。
- 设置页新增 INFO、WARN、ERROR、CRITICAL、DEBUG 五档日志等级。
- INFO 保持旧 C++ 管理器样式，详细 Rust 诊断仅在 DEBUG 显示。
- 清空日志后不再从运行时缓冲恢复旧记录。
- 修复设置写入和多个页面并发请求的响应串线。
- 修复配置页重建时的空配置保存风险和非法模式归一化。
- 修复共享 UID 枚举顺序变化、进程退出后的幽灵等待状态与并发日志覆盖。
- 安装器不再卸载 NoANR；管理器覆盖失败会中止模块升级。
- 正式发布增加同次构建会话、APK 签名证书和来源证明门禁。
