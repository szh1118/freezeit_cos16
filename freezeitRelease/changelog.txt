### v3.3.4SelfUse

1. 修复 Binder 调试节点不可读时 `binder_queue unknown` 被误判为活动事务，导致所有候选应用永久停在等待冻结状态；未知信息仅保留为诊断，明确 active/busy 事务仍会延迟冻结。
2. 修复 Binder freezer 降级后的 SIGSTOP 回退操作被记录成 `cgroup.freeze` 的问题，日志现在准确显示 `signal.stop`。
3. 强化控制循环的冻结/解冻回收、重启恢复、前台保护、共享 UID 和系统 UID 防护，减少应用状态长期失真或错误控制。
4. 加固 manager socket、Xposed bridge 与配置同步的帧长度、时限、会话一致性和失败降级路径。
5. 修复 hook 侧前台、待冻结、WakeLock、Alarm、ANR、广播与 Athena 处理中的状态边界问题。
6. Android 16 上未实现的 `BroadcastQueueModernImpl` 广播抑制改为可选降级，不再让核心冻结/解冻控制整体停用；该广播拦截优化暂不可用。
7. 修复管理器配置、缓存、网络、内容 URI 和页面并发更新中的资源泄漏、错误响应与 UI 回归。
8. 正式发布链增加 APK 证书、构建会话、来源证明和 ZIP 校验覆盖。

### v3.3.3SelfUse

1. 修复 manager socket 在校验长度前按客户端声明的 payload_len 分配内存且无读取超时的问题；恶意或畸形长度声明不再触发 ~4GB 分配或永久阻塞控制循环。
2. 守护进程接受循环增加 catch_unwind，单个请求的 panic 不再拖垮整个 daemon，避免经 SIGSTOP 冻结的应用因 daemon 退出而无人恢复。
3. 守护进程重启时扫描受管 uid 中处于停止状态（T/t）的非 cgroup 进程并补发 SIGCONT，修复上一轮用信号冻结的应用在重启后永久卡死的问题。
4. 修复 `can_freeze` 在仅有 cgroup freezer、无 binder freezer 时仍路由到 Freeze 路径导致 binder 前置失败回滚、失去 cgroup.freeze 精确冻结的问题。
5. 把控制循环的 hook 健康检查与配置同步的 xposed bridge 网络往返移出全局 state Mutex 持有期；移除管理器请求路径的冗余每请求刷新，慢 hook 桥接不再阻塞冻结/解冻。
6. 修复 FreezeitService Xposed 桥接 `acceptHandle` 未捕获 IOException 的问题；客户端中途断开不再泄漏 client socket、不再消耗重试次数导致整个启动周期桥接死亡。
7. 修复 Java 客户端 `payloadLen` 无上限校验导致畸形响应头触发 OutOfMemoryError 崩溃的问题。
8. 修复 Config 页下拉刷新在后台线程调用 `requireContext()` 导致 Fragment detach 后崩溃的问题。
9. 修复 ClearLog 返回空 Vec 使 Logcat 无法清屏、GetProcState 受日志级别过滤导致 check 按钮无响应的问题。
10. `set_settings_var` 改用原子写（临时文件+fsync+rename），避免断电时 settings.db 损坏。
11. `escape_json` 按 RFC 8259 转义 U+0000..U+001F 控制字符，避免 package_name 含裸控制字符破坏 operation log JSON。
12. 安装器暂存 APK 权限由 0666 收紧为 0600，避免任意应用在 pm install 前篡改 APK。

### v3.3.2SelfUse

1. 修复 Manager 无操作保存时把展示默认项写入配置的问题。
2. 保留 `appcfg.txt` 原有包名、顺序、注释、空行与未修改字节；文件缺失时安全重建。
3. 实现 daemon 命令 11 的冻结状态快照，恢复 Manager 冻结状态页。
4. 对相同共享 UID 进程集合的安全拒绝去重，避免日志每秒刷屏。
5. 下载延迟持续续期时只记录首次事件，避免同一下载状态反复刷屏。
6. 修正控制事务错误文案，不再把 Binder/cgroup 失败统一写成 signal 失败。
7. 清空工作日志前增加明确确认框，取消操作不会改动日志。
8. 用五档日志等级替代“调试日志”开关：INFO、WARN、ERROR、CRITICAL、DEBUG。
9. INFO 严格恢复旧 C++ 管理器格式；Rust 运行时、UID/PID、后端、结果和原因仅在 DEBUG 中显示。
10. 修复清空日志后旧操作从运行时环形缓冲再次出现的问题。
11. 修复设置页异步响应串线、普通控件写入失败不回滚，以及多个页面共享响应缓冲导致的内容覆盖。
12. 修复配置页重建后可能发送空或残缺配置、非法冻结模式未归一化的问题。
13. 修复控制线程快照覆盖并发日志、共享 UID 枚举顺序变化重复告警、进程退出后残留幽灵等待状态。
14. 安装器不再静默卸载 NoANR；管理器覆盖失败时中止模块升级，避免新守护进程与旧管理器混用。
15. 正式发布增加同次构建会话、APK 签名证书和 provenance v2 门禁。
